Critical Findings (MUST address before implementation)

1. Incorrect Reverse Relationship Pattern for linkParent

- Location: v2.md lines 456-461, 463-467
- Evidence:
    - Found in Advanced/Resources/metadata/entityDefs/WorkflowLogRecord.json lines 6-31: linkParent field has entityList, belongsToParent link does NOT have entityList
    - Found in Advanced/Resources/metadata/entityDefs/BpmnProcess.json lines 38-44 and 203-206: same pattern - linkParent field has entityList, belongsToParent link does not
- Assumption: The scope assumes procedure entities (ProcedimentoConsulta, etc.) need hasMany links to sessoes/agendamentos, and Sessao's belongsToParent link should have entityList
- Risk: Implementation will create incorrect relationship structures. EspoCRM's linkParent pattern does NOT use hasMany on parent entities - the polymorphic relationship is queried differently. Adding entityList to belongsToParent links is not the correct pattern.
- Remedy:
    - REMOVE the hasMany sessoes/agendamentos links from all 5 procedure entities (lines 456-461)
    - REMOVE entityList from belongsToParent links (Sessao, Agendamento, etc.)
    - Correct pattern:
      // In Sessao entityDefs:
      fields: {
      procedimento: {
      type: linkParent,
      entityList: [ProcedimentoConsulta, ProcedimentoInjetavel, ProcedimentoImplante, ProcedimentoEstetico, ProcedimentoAtividadeFisica]
      }
      },
      links: {
      procedimento: {
      type: belongsToParent
      // NO entityList here - it's on the field, not the link
      }
      }
        - Parent entities (ProcedimentoConsulta, etc.) do NOT need reverse hasMany links for linkParent children

2. Missing Link Index Specifications for All linkParent Fields

- Location: v2.md lines 499-504 (mentions pattern but incomplete)
- Evidence: Standard EspoCRM pattern requires composite indexes for linkParent fields (found in core documentation patterns)
- Assumption: The scope assumes indexes are mentioned but doesn't list them explicitly for each entity
- Risk: Database queries on linkParent fields will be slow; missing unique constraint enforcement
- Remedy: Add explicit index specifications to each entity with linkParent:
  indexes: {
  procedimento: {
  columns: [procedimentoType, procedimentoId]
  }
  }
  Required for: ConvenioRegra, TabelaDePrecos, ProgramaItem, MovimentacaoEstoque, PrescricaoItem, Documento, Sessao, Agendamento, ProcedimentoRealizado, OrcamentoItem, LancamentoFinanceiro (11 entities total)
