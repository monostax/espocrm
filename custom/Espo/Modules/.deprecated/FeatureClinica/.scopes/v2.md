The manifest uses "Saude" as the module name, but the implementation should be on "FeatureClinica" module. All file paths need to be updated accordingly.

Implementation Recommendations

1. Phase 1 Priority: Start with Unidade, Profissional, and all 5 Procedimento\* entities. These are foundational and have no dependencies on other custom entities.
2. linkParent Pattern: All linkParent fields need proper indexes:
   indexes: {
   procedimento: {
   columns: [procedimentoType, procedimentoId]
   }
   }
3. Cascade Delete: Use the existing Global\Hooks\Common\CascadeDelete by adding cascadeDelete configuration to entityDefs instead of writing custom hooks.
4. ACL Strategy: Define consistent ACL patterns:
    - Administrative entities (Unidade, Convenio): admin for create/delete, team for read
    - Clinical entities: owner for edit, team for read
    - Financial entities: admin for delete, owner for edit
5. Testing Order: Test linkParent fields first - they are the most complex and error-prone part of this implementation.
